# Frontend Design — навык дизайна интерфейсов

> Создание визуально отличительных, production-grade веб-интерфейсов с высоким качеством дизайна.

## Когда активировать

Когда нужно создать: веб-страницу, компонент, лендинг, дашборд, UI-элемент.

## Перед кодом — определи

1. **Цель** — что делает интерфейс и для кого?
2. **Тональное направление** — минимализм, максимализм, ретро, футуризм?
3. **Технические ограничения** — фреймворк, размер экрана, браузеры
4. **Что сделает запоминающимся** — одна деталь, которая отличает от шаблона

## Фокус-области

### Типографика
- Уникальные, выразительные шрифты. Не Arial, не Inter по дефолту.
- Иерархия: заголовок > подзаголовок > текст — каждый уровень отличим

### Цвет
- Смелые, связные палитры через CSS-переменные
- Стратегические акценты — не всё подряд цветное
- Тёмная тема — не инверсия, а отдельная продуманная схема

### Движение
- Анимации по контексту: появление, hover, переходы
- Не анимируй ради анимации — каждое движение = смысл

### Пространство
- Асимметрия, наложения, ломка сетки — но с намерением
- Белое пространство как дизайн-элемент

### Атмосфера
- Градиенты, текстуры, тени, пользовательские детали
- Глубина и объём через слои

## Чего избегать

- Дефолтные шрифты без обоснования
- Предсказуемые фиолетовые градиенты
- Шаблонные компоненты без характера
- «Дизайн как у всех» — без контекстной привязки

## Принцип

Намеренность важнее интенсивности. И минимализм, и максимализм работают — когда за ними стоит осознанное решение.
