# Full-Stack Developer — навык разработки

> Полный цикл веб-разработки: от базы данных до деплоя.

## Когда активировать

Когда нужно написать код: бэкенд, фронтенд, API, базу данных, деплой.

## Стек

- **Frontend:** React, Next.js, TypeScript, Tailwind CSS
- **Backend:** Node.js, Express/Fastify
- **Данные:** PostgreSQL, MongoDB, Prisma ORM, Redis
- **DevOps:** Vercel, Docker, GitHub Actions

## Структура проекта

### Frontend
```
src/
├── components/    — UI-компоненты
├── hooks/         — кастомные хуки
├── types/         — TypeScript-типы
├── utils/         — утилиты
├── pages/         — страницы (Next.js)
└── styles/        — стили
```

### Backend
```
src/
├── routes/        — маршруты API
├── controllers/   — логика обработки
├── models/        — модели данных
├── middleware/     — промежуточное ПО
├── utils/         — утилиты
└── config/        — конфигурация
```

## Принципы

- **Компоненты маленькие и фокусированные.** Один компонент = одна задача.
- **TypeScript везде.** Типизация — не опция, а стандарт.
- **RESTful API.** Правильные HTTP-методы и статус-коды.
- **Валидация на входе.** Zod для схем, санитизация данных.
- **Разделение ответственности.** Роут не знает про базу. Контроллер не знает про HTTP.

## Безопасность

- Валидируй все входные данные
- Санитизируй пользовательские данные
- Параметризованные запросы (не строковая конкатенация)
- Rate limiting на публичных эндпоинтах
- CORS настроен правильно

## Производительность

- Code splitting и lazy loading
- Оптимизация бандла
- Кэширование (Redis, HTTP-кэш)
- Индексы в базе данных
