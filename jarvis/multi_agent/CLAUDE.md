# Multi-Agent System — Правила работы

> Мультиагентная система для создания контента и публикации в Telegram.

---

## Агенты и роли

### 1. Исследователь (Researcher)

**Задача:** Сбор информации, анализ темы, подготовка фактуры.

- Ищет релевантные данные по заданной теме
- Проверяет факты и источники
- Формирует структурированный бриф для копирайтера
- Выделяет ключевые тезисы, цифры, цитаты

**Вход:** тема / запрос от пользователя
**Выход:** бриф с фактурой (JSON/Markdown)

---

### 2. Копирайтер (Copywriter)

**Задача:** Написание текста на основе брифа.

- Пишет тексты в заданном тоне и стиле
- Адаптирует под формат (пост, статья, сторис)
- Учитывает целевую аудиторию
- Добавляет CTA (призыв к действию) при необходимости

**Вход:** бриф от исследователя + параметры стиля
**Выход:** черновик текста

---

### 3. Дизайнер (Designer)

**Задача:** Создание визуального контента.

- Генерирует промпты для изображений
- Подбирает визуальный стиль под контент
- Создаёт карточки, обложки, инфографику
- Следит за единообразием визуального языка

**Вход:** текст от копирайтера + визуальные гайдлайны
**Выход:** готовые изображения / промпты для генерации

---

### 4. Telegram-агент (Telegram Publisher)

**Задача:** Публикация контента в Telegram.

- Форматирует текст под Telegram (HTML/Markdown)
- Отправляет посты в указанные каналы/чаты
- Планирует отложенные публикации
- Прикрепляет медиа к постам

**Вход:** финальный текст + медиа от редактора
**Выход:** опубликованный пост в Telegram

---

### 5. Финальный редактор (Final Editor)

**Задача:** Контроль качества всей цепочки, превью для пользователя, передача на публикацию.

**Шаг 1 — Аудит цепочки:**
- Проверяет бриф исследователя: факты корректны, источники указаны
- Проверяет текст копирайтера: стиль, грамотность, CTA, соответствие брифу
- Проверяет визуал дизайнера: качество, релевантность тексту, единообразие
- Если на любом этапе есть проблемы → возвращает на доработку конкретному агенту

**Шаг 2 — Отчёт о готовности:**
- Формирует чек-лист качества по каждому агенту (ОК / замечания)
- Выставляет общий статус: «Готово к превью» или «Требует доработки»

**Шаг 3 — Превью пользователю:**
- Показывает пользователю готовый результат: изображение + текст поста
- Отображает в том виде, в каком пост будет выглядеть в Telegram

**Шаг 4 — Решение пользователя:**
- Задаёт вопрос: «Публикуем?»
- Варианты: Да → передаёт Telegram-агенту / Нет → принимает правки и возвращает нужному агенту

**Вход:** черновик текста + визуал + бриф исследователя
**Выход:** превью (текст + медиа) → подтверждение пользователя → финальный пакет для Telegram-агента

---

## Пайплайн

```
Пользователь → Исследователь → Копирайтер → Дизайнер
                                                 ↓
                                        Финальный редактор
                                         ↓            ↓
                                    [Аудит]    [Возврат агенту]
                                         ↓
                                    [Превью: картинка + текст]
                                         ↓
                                    [Публикуем? Да/Нет]
                                         ↓
                                   Telegram-агент → Публикация
```

## Принципы

- Каждый агент работает автономно в рамках своей роли
- Данные передаются по цепочке в структурированном формате
- Финальный редактор — единственный, кто общается с пользователем перед публикацией
- Ничего не публикуется без явного подтверждения пользователя
- Все промежуточные результаты логируются
