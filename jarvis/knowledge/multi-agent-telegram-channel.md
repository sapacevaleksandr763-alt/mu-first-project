> Архитектура системы из 6 агентов для автоматизации контент-маркетинга в Telegram.

## 6 агентов: роли и функции

### 1. 🎬 РЕЖИССЁР (Orchestrator Agent)

**Роль:** Главный координатор. Управляет всеми агентами, распределяет задачи, контролирует качество, отчитывается владельцу.

**Входные данные:**
- Тема канала / ниша (от владельца)
- Результаты работы других агентов
- Метрики из Аналитика

**Что делает:**
- Формулирует задания для каждого агента на неделю
- Проверяет контент перед публикацией (качество, тон, соответствие стратегии)
- Принимает решение: опубликовать / доработать / отклонить
- Генерирует еженедельный отчёт владельцу
- Корректирует стратегию на основе данных Аналитика

**Выходные данные:**
- Задания агентам (JSON-формат)
- Еженедельный отчёт (markdown)
- Решения по контенту (approve/revise/reject)

**Инструменты:**
- Telegram Bot API (отправка отчётов владельцу)
- Внутренняя шина сообщений (координация агентов)

---

### 2. 🔍 СКАУТ (Research Agent)

**Роль:** Поиск информации в интернете. Находит статьи, видео, изображения, обсуждения, тренды по теме канала.

**Входные данные:**
- Тема / ключевые слова от Режиссёра
- Список источников (сайты, каналы, блоги)
- Запрос на конкретный тип контента (видео, статья, инфографика)

**Что делает:**
- Парсит целевые сайты, блоги, форумы
- Мониторит конкурентные Telegram-каналы
- Ищет в YouTube, Google, Reddit, VK
- Собирает отзывы, комментарии, обсуждения
- Фильтрует по релевантности и свежести
- Формирует «дайджест находок» с ссылками и ключевыми тезисами

**Выходные данные:**
- Дайджест находок (JSON):
  ```json
  {
    "topic": "...",
    "date": "2026-07-15",
    "items": [
      {
        "type": "article|video|image|discussion",
        "title": "...",
        "url": "...",
        "summary": "3-5 предложений",
        "key_points": ["...", "..."],
        "relevance_score": 0.85
      }
    ]
  }
  ```

**Инструменты:**
- Web scraping (Playwright / Puppeteer)
- Google Custom Search API
- YouTube Data API
- Telegram API (чтение публичных каналов)
- RSS-парсер

---

### 3. ✍️ АВТОР (Content Agent)

**Роль:** Создаёт контент на основе данных Скаута. Пишет посты, статьи, сценарии для роликов, промпты для изображений.

**Входные данные:**
- Дайджест находок от Скаута
- Контент-план от Планера (тип поста, тема, дедлайн)
- Tone of voice и стиль канала (из конфига)
- Фидбэк от Режиссёра (если доработка)

**Что делает:**
- Пишет посты для Telegram (короткие 200-500 символов, длинные 1000-2000)
- Создаёт статьи (Telegraph / Telegra.ph)
- Пишет сценарии для коротких видео (30-60 сек)
- Генерирует промпты для Дизайнера (обложки, карточки)
- Составляет CTA (call to action) и хуки для вовлечения
- Создаёт воронку вовлечения: закреп → приветственный пост → серия вводных постов

**Выходные данные:**
- Готовый контент (markdown):
  ```json
  {
    "type": "post|article|video_script",
    "title": "...",
    "body": "...",
    "cta": "...",
    "hashtags": ["...", "..."],
    "visual_prompt": "описание картинки для Дизайнера",
    "scheduled_date": "2026-07-16",
    "status": "draft"
  }
  ```

**Инструменты:**
- LLM (Claude API) для генерации текстов
- Telegra.ph API (публикация статей)

---

### 4. 🎨 ДИЗАЙНЕР (Visual Agent)

**Роль:** Создаёт визуальный контент — обложки постов, карточки с цитатами, инфографику, превью для видео.

**Входные данные:**
- Visual prompt от Автора
- Брендбук канала (цвета, шрифты, стиль)
- Тип визуала (обложка, карточка, инфографика)

**Что делает:**
- Генерирует изображения через AI (DALL-E / Midjourney API / Flux)
- Создаёт текстовые карточки (HTML → PNG через Puppeteer)
- Делает инфографику по данным
- Собирает превью для видео
- Применяет брендинг канала (водяной знак, цвета, лого)

**Выходные данные:**
- Файлы изображений (PNG/JPG)
- Метаданные: размер, тип, для какого поста

**Инструменты:**
- Image generation API (DALL-E 3 / Flux)
- Puppeteer (HTML → PNG для карточек)
- Sharp (обработка изображений в Node.js)

---

### 5. 📅 ПЛАНЕР (Scheduler Agent)

**Роль:** Составляет контент-план на месяц, управляет расписанием публикаций, отправляет контент в канал по графику.

**Входные данные:**
- Утверждённый контент от Режиссёра
- Метрики от Аналитика (лучшее время для постов)
- Праздники, события, тренды (календарь)

**Что делает:**
- Составляет контент-план на 4 недели (4 поста/неделю = 16 постов/месяц)
- Распределяет типы контента по дням:
  ```
  Пн — информационный пост (статья/разбор)
  Ср — визуал (карточка/инфографика)
  Пт — видео/reels (короткий ролик)
  Вс — вовлечение (опрос/обсуждение/CTA)
  ```
- Определяет оптимальное время публикации
- Автоматически публикует по расписанию через Telegram Bot API
- Ведёт очередь контента (буфер на 1-2 недели вперёд)

**Выходные данные:**
- Контент-план (JSON/markdown таблица)
- Лог публикаций (что, когда, статус)

**Инструменты:**
- Telegram Bot API (публикация)
- Cron / node-schedule (расписание)
- Calendar API (события, праздники)

---

### 6. 📊 АНАЛИТИК (Analytics Agent)

**Роль:** Собирает метрики канала, анализирует вовлечённость, формирует еженедельный отчёт с рекомендациями.

**Входные данные:**
- Статистика канала (Telegram API)
- Лог публикаций от Планера
- Метрики постов: просмотры, реакции, комментарии, репосты, отписки

**Что делает:**
- Собирает метрики каждого поста через 24ч и 72ч после публикации
- Считает ключевые показатели:
  - ERR (Engagement Rate by Reach) = (реакции + комментарии + репосты) / просмотры
  - Рост подписчиков (нетто: пришли − ушли)
  - Лучшее время для постов (по вовлечённости)
  - Топ-3 поста недели
  - Худший пост недели (почему не зашёл)
- Формирует еженедельный отчёт:
  ```
  ## Отчёт за неделю 14-20 июля 2026
  
  Подписчики: 1,250 (+45)
  Посты: 4
  Средний ERR: 4.2%
  Лучший пост: "..." (ERR 8.1%)
  
  ### Что сработало
  - ...
  
  ### Что улучшить
  - ...
  
  ### Рекомендации на следующую неделю
  - ...
  ```
- Выявляет тренды: какие темы/форматы растут, какие падают
- Рекомендует корректировки стратегии

**Выходные данные:**
- Еженедельный отчёт (markdown)
- Рекомендации для Режиссёра (JSON)
- Дашборд метрик (опционально, HTML)

**Инструменты:**
- Telegram Bot API (getUpdates, getChatMembersCount)
- Telegram Channel Stats (TGStat API, опционально)
- Хранилище метрик (SQLite / JSON-файлы)

## Цикл работы (еженедельный)

```
День 1 (Пн): СКАУТ ищет материалы → АВТОР получает дайджест
День 2 (Вт): АВТОР пишет контент на неделю → ДИЗАЙНЕР делает визуалы
День 3 (Ср): РЕЖИССЁР проверяет всё → approve/revise
День 3-7:    ПЛАНЕР публикует по расписанию (Пн, Ср, Пт, Вс)
День 7 (Вс): АНАЛИТИК собирает метрики → отчёт РЕЖИССЁРУ → отчёт владельцу
```

## Ежемесячный цикл

```
Неделя 1-3: Стандартный еженедельный цикл
Неделя 4:   АНАЛИТИК → месячный отчёт
             РЕЖИССЁР → корректировка стратегии
             ПЛАНЕР → контент-план на следующий месяц
```

## Порядок разработки

1. **Фаза 1 — Ядро (1-2 недели)**
   - Шина сообщений + очередь задач
   - Базовый Режиссёр (раздача задач, проверка)
   - Подключение Telegram Bot API

2. **Фаза 2 — Контент (1-2 недели)**
   - Скаут: поиск + парсинг
   - Автор: генерация постов через Claude API
   - Планер: расписание + автопубликация

3. **Фаза 3 — Визуал (1 неделя)**
   - Дизайнер: генерация изображений
   - Шаблоны карточек

4. **Фаза 4 — Аналитика (1 неделя)**
   - Аналитик: сбор метрик
   - Еженедельные отчёты
   - Рекомендации

5. **Фаза 5 — Воронка (1 неделя)**
   - Лендинг канала
   - Приветственная серия постов
   - CTA и механики вовлечения

**Общий срок: 5-7 недель до MVP**
