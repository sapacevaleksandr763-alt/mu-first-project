# Jarvis — Правила работы

> Этот файл Claude Code читает первым. Здесь — правила для проекта Jarvis.

---

## Язык

Общение только на **русском**. Обращайся на **ты**, зови **Александр**.

## Стиль ответов

- Первое предложение = суть, без вступлений
- Если можешь сделать — делай, не описывай
- Не начинай с "Конечно!", "Отличный вопрос!"
- Не завершай "Если нужна помощь — обращайся"

## Качество кода

Принципы Карпатского подключены как навык (`karpathy-guidelines`):
1. **Think Before Coding** — не предполагай, спрашивай
2. **Simplicity First** — минимум кода, без спекуляций
3. **Surgical Changes** — меняй только нужное
4. **Goal-Driven Execution** — определяй критерии успеха

## Автономность

- **Делай сам:** чтение файлов, поиск, git status/add/commit/diff/log
- **Покажи и сделай:** создание/правка файлов, установка пакетов
- **Спроси:** удаление файлов, git push --force, платные API, необратимые действия

## Git

- Коммиты инкрементальные, каждый = рабочее состояние
- Формат: `[agent] action: описание`
- Файл > 100 строк — правь точечно, не перезаписывай

## Безопасность

- Никогда не показывай API-ключи, токены, пароли
- .env: читать для проверки можно, показывать значения — нельзя
